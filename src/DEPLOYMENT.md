## ğŸ“¦ Exporting for External Deployment

If deploying outside Figma Make:

### 1. Clone the Project
```bash
# Copy all files from Figma Make
# Ensure you have:
- /components (all user screens)
- /components/admin (admin screens)
- /components/ui (ShadCN components)
- /utils (API and locales)
- /supabase/functions/server (backend)
- /styles/globals.css
- App.tsx, main.tsx
```

### 2. Set Up Supabase
```bash
# Install Supabase CLI
npm install -g supabase

# Initialize Supabase
supabase init

# Link to your project
supabase link --project-ref your-project-ref

# Deploy edge functions
supabase functions deploy make-server-4192bdc4
```

### 3. Configure Environment Variables

Create `.env` file:
```bash
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key

# In Supabase Dashboard > Edge Functions > Secrets:
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
GEMINI_API_KEY=your-gemini-api-key
```

### 4. Deploy Frontend

**Vercel:**
```bash
npm install -g vercel
vercel --prod
```

**Netlify:**
```bash
npm run build
# Drag /dist folder to Netlify
```

### 5. Update API Endpoints
In `/utils/supabase/info.tsx`, update:
```typescript
export const projectId = "your-project-id";
export const publicAnonKey = "your-anon-key";
```
